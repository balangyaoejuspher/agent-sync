When adding a new endpoint:

1. Decide explicitly whether it should be authenticated.
2. If yes, reuse the most common middleware/decorator from section 1.
3. If a role is required, use one of the constants from section 2.
4. Never silently expose data that was previously gated.

---

## 6. Caveats

- This skill uses heuristic pattern matching. It will produce false positives (routes flagged as public when they actually inherit auth from a parent router) and false negatives (custom auth wrappers we don't recognize). Always cross-check with the actual handler code.
- Re-run after middleware refactors:

```bash
npx agent-sync add auth-map
```
