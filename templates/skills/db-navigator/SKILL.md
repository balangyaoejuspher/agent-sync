## 2. How to add a column safely

Follow the convention this repository already uses with **{{DB_DIALECT}}**. Generic guidance:

1. Locate the relevant model in `references/schema.*`.
2. Add the field with an explicit type and (where supported) a default for backfill safety.
3. Generate a migration using the project's package manager (`{{PROJECT_PACKAGE_MANAGER}}`).
4. Review the generated migration for destructive operations (column drops, type narrowing, non-null without default).
5. Update any seed / factory / fixture files that construct the model.
6. Update relevant TypeScript / Python types if they are not auto-generated.

Refuse to write a migration that:

- Drops a column referenced by the patterns listed in section 3, unless the user has explicitly removed those callsites.
- Renames a primary key.
- Changes a column type without a backfill plan.
