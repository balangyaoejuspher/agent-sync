## 2. How routes are organized in this codebase

- **Framework**: `{{PROJECT_FRAMEWORK}}`
- **Routing style**: `{{ROUTING_STYLE}}`
- **Total endpoints captured**: `{{API_ROUTE_COUNT}}`

When asked to add an endpoint:

1. Match the file/module layout you see above. Place the new handler next to its siblings; do not invent a parallel structure.
2. Reuse existing middleware, validation, and error handling — search for the closest neighbour first.
3. Follow the path-naming convention already in use (kebab-case vs camelCase, plural vs singular, version prefix or not).
4. If authentication is enforced via a shared middleware (`requireAuth`, `@login_required`, `before_action :authenticate`), include it on the new route unless the user explicitly says public.
