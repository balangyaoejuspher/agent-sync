## 2. Conventions

When modifying or adding a pipeline:

1. Match the existing workflow's structure (job names, runner OS, action versions). Don't introduce a new style alongside the current one.
2. Pin third-party Actions to a commit SHA when the existing workflows do — never downgrade pinning.
3. If you add a new required check, surface it explicitly in the PR so branch-protection rules can be updated.
4. Re-use existing reusable workflows / composite actions before creating new ones.
5. Don't list a path-filtered workflow as a required status check — skipped checks stay pending forever and block PRs.
